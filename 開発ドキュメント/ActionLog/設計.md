# ActionLog 設計

## 1. DB テーブル設計

### action_log（1行 = 1アクション）

```sql
CREATE TABLE action_log (
  id             bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  ticker         text NOT NULL,
  action_date    date NOT NULL,
  archive_id     text,
  action_text    text,
  story          text,
  decision       text,
  quantity       integer DEFAULT 0,
  price          numeric,
  money_in       numeric DEFAULT 0,
  cumulative_invested numeric DEFAULT 0,
  total_assets   numeric,
  pnl            numeric DEFAULT 0,
  is_auto        boolean DEFAULT true,
  user_overrides jsonb DEFAULT '{}',
  created_at     timestamptz DEFAULT now(),
  updated_at     timestamptz DEFAULT now(),
  UNIQUE(ticker, archive_id)
);

CREATE INDEX idx_action_log_ticker_date ON action_log (ticker, action_date);
```

| カラム | 説明 |
|--------|------|
| `archive_id` | 元 archive レコードの ID。手動追加行は NULL |
| `action_text` | "30,000円買った" 等の短文 |
| `story` | beginner_summary テキスト |
| `money_in` | この行の投入額（買い=正、売り=負、保留=0） |
| `cumulative_invested` | 入れたお金の累計（この行時点） |
| `total_assets` | その銘柄の評価額 |
| `pnl` | 損益 = total_assets - cumulative_invested |
| `is_auto` | 自動投入か手動追加か |
| `user_overrides` | ユーザーが手動で直したフィールドの一覧。例：`{"action_text": true, "money_in": true}` → この2つは自動上書きしない |

### action_log_handoff（1行 = 1銘柄1か月）

```sql
CREATE TABLE action_log_handoff (
  id             bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  ticker         text NOT NULL,
  year_month     text NOT NULL,
  handoff_text   text,
  generated_at   timestamptz,
  created_at     timestamptz DEFAULT now(),
  UNIQUE(ticker, year_month)
);
```

## 3. 自動計算ロジック

### 依存関係

```
money_in（この行の投入額）
    ↓
cumulative_invested = 前行の cumulative_invested + この行の money_in
    ↓
pnl = total_assets - cumulative_invested
```

`total_assets` は独立値（ユーザー入力 or 自動値）。

### money_in 編集時

1. 編集行の `cumulative_invested` を再計算
2. 編集行の `pnl` を再計算
3. **後続の全行**に連鎖的に波及：各行の `cumulative_invested` → `pnl` を上から順に再計算
4. まとめて DB を更新

### total_assets 編集時

1. その行の `pnl` のみ再計算：`total_assets - cumulative_invested`
2. 他の行には影響しない

### total_assets の自動値

- 自動投入時：`archive の current_price × その銘柄の保有株数累計`
- 保有株数累計 = 過去の全 action_log の `quantity` を合算（BUY/ADD は正、SELL/REDUCE は負）

## 4. データフロー

### 自動投入フロー（Watch 完了時）

```
Watch/src/main.py の process_one_ticker():
  ...
  await notify(payload)         # ← beginner_summary がここで生成
  # ★ 以下を追加 ★
  await insert_action_log(
      ticker=ticker,
      archive_id=archivelog_id,
      newplan_full=newplan_full,
      beginner_summary=payload.beginner_summary,
  )
  safe_db(update_archivelog, archivelog["id"], active=False)
```

### insert_action_log の処理

```
1. newplan_full の YAML データを読み取り
2. decision.final を見て action_text を自動生成（例：BUY → "○○円買った"）
3. allocation_jpy を money_in にセット（SELL/REDUCE のときはマイナスにする）
4. 既存行から前行の cumulative_invested を取得
5. cumulative_invested = prev + money_in
6. 保有株数累計を計算 → total_assets = price × 累計株数
7. pnl = total_assets - cumulative_invested
8. action_log テーブルに保存（同じ archive からの二重投入は DB が自動で防ぐ）
```

### 引き継ぎ文の生成フロー

```
1. ユーザーが月ナビで別の月に移動
2. action_log_handoff テーブルにその (ticker, year_month) があるか確認
3. なし → 前月の action_log 全行を取得 → Opus に渡して引き継ぎ文生成 → DB 保存
4. あり → 保存済みのテキストをそのまま表示
```
