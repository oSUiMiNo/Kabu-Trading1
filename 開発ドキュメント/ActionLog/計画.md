# ActionLog 実装計画

## 実装ステップ

| # | 内容 | 完了条件 |
|---|------|---------|
| 1 | DB にテーブル作成 | Supabase に `action_log` と `action_log_handoff` テーブルが作成されている |
| 2 | ActionLog/ フォルダ作成 + 仮想環境構築 | `pyproject.toml` + `uv sync` + `.venv` 構築完了 |
| 3 | `shared/supabase_client.py` に読み書き関数を追加 | action_log / handoff の追加・取得・更新の関数が動作 |
| 4 | `calc_engine.py` | 連鎖再計算の単体テスト合格 |
| 5 | `auto_populate.py` | archive → action_log 変
換の単体テスト合格 |
| 6 | `data_service.py` | DB CRUD の結合テスト合格 |
| 7 | `main.py`（NiceGUI UI） | ブラウザ表示、月ナビ、セル編集、自動再計算が動作 |
| 8 | Watch 連携 | Watch 実行後に action_log にレコードが投入される |
| 9 | `handoff_service.py` + サブエージェント | 月切り替え時に引き継ぎ文が生成・表示される |
| 10 | 一気通貫テスト | Watch → DB → 画面 の一連フローが正常動作 |

## 依存関係

```
Step 1（DB にテーブル作成）
  → Step 2（フォルダ作成）
    → Step 3（DB 読み書き関数）
      → Step 4, 5, 6（計算・変換ロジック。同時に進められる）
        → Step 7（画面）
          → Step 8（Watch 連携）
          → Step 9（引き継ぎ文）
            → Step 10（一気通貫テスト）
```
