## 変更ログ

### Step 1: DB マイグレーション
- Supabase MCP の `apply_migration` で `action_log` と `action_log_handoff` テーブルを作成
- `list_tables` と `SELECT` で作成を確認済み

### Step 2: ディレクトリ骨格 + venv
- `ActionLog/src/` に pyproject.toml、AgentUtil.py、main.py（最小起動確認用）、各プレースホルダーを作成
- `ActionLog/.claude/CLAUDE.md` と `commands/handoff-generator.md`（予約）を作成
- `uv sync` で NiceGUI 3.9.0 含む68パッケージをインストール

### Step 3: shared CRUD 追加
- `shared/supabase_client.py` に action_log 用6関数 + handoff 用2関数を追加
  - `create_action_log`, `list_action_logs`, `list_all_action_logs`, `update_action_log`, `get_latest_action_log`, `list_action_log_archive_ids`
  - `get_action_log_handoff`, `upsert_action_log_handoff`
- 空テーブルクエリ、INSERT、SELECT、UPDATE、二重投入防止（UNIQUE制約）を全て動作確認

### Step 4: calc_engine.py
- `ActionLog/src/calc_engine.py` に5関数を実装
  - `recalculate_all`, `recalculate_from`, `calc_pnl`, `calc_total_shares`, `calc_total_assets`
- 7テストケース通過（正常系、連鎖再計算、None処理、空リスト、SELL減算、非破壊コピー）

### Step 5: auto_populate.py
- `ActionLog/src/auto_populate.py` に4関数を実装
  - `parse_newplan_full`, `build_action_text`, `build_action_log_row`, `populate_from_archive`
- 9単体テスト通過（BUY/HOLD/NO_BUY/SELL/ADD変換、不正YAML処理）
- DB 投入テスト通過（投入成功 + 二重投入防止）
- 修正：不正 YAML 入力時の例外処理を追加（`yaml.YAMLError` をキャッチ）

### Step 6: data_service.py
- `ActionLog/src/data_service.py` に5関数を実装
  - `get_monthly_data`, `handle_edit`, `populate_existing_archives`, `get_ticker_list`, `get_available_months`
- 7結合テスト通過
  - 空テーブル、money_in 連鎖再計算、total_assets 編集→pnl再計算、action_text 編集、user_overrides 記録、月一覧取得、銘柄一覧取得

### Step 7: main.py（NiceGUI UI）
- `ActionLog/src/main.py` を全面実装
  - `/` ルートページ：銘柄選択ドロップダウン + 既存 archive 取り込み機能
  - `/ticker/{ticker}` アクションログページ：月ナビ + AG Grid テーブル
  - AG Grid：autoHeight + wrapText（ストーリー列可変行高）、通貨フォーマット（formatJPY）、損益色分け（正=緑、負=赤）
  - 引き継ぎ文：テーブル先頭行に `is_handoff` フラグ付きで挿入、編集不可
  - cellValueChanged イベントで data_service.handle_edit を呼び出し → グリッド再描画
- テストデータ（DEMO 銘柄 3行 + 引き継ぎ文）を投入して構文・データ取得を確認済み
- 実際のブラウザ表示は `uv run python main.py` で起動して確認

### Step 8: Watch 連携
- `Watch/src/main.py` の `notify(payload)` 直後に action_log 投入コードを追加
  - `try/except` で全体を囲み、ActionLog ブロックが存在しない環境でも Watch 本体に影響しない
  - `sys.path` に ActionLog/src を動的追加、import も try 内で実行
  - `payload.beginner_summary` をストーリーとして action_log に保存
- 構文チェック通過

### Step 9: handoff_service.py + サブエージェント
- `ActionLog/.claude/commands/handoff-generator.md` を実装（claude-opus-4-6、2〜3文、150文字以内）
- `ActionLog/src/handoff_service.py` に3関数を実装
  - `get_cached_handoff`：DB キャッシュのみ同期チェック
  - `generate_handoff`：前月データ → Opus で引き継ぎ文生成
  - `get_or_generate_handoff`：キャッシュあれば返す、なければ生成して保存
- `main.py` の `refresh_grid` を更新：キャッシュ即時表示 + キャッシュミス時はバックグラウンド生成
- キャッシュ取得・プロンプト構築の動作確認済み
