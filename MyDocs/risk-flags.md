# リスクフラグ

## 概要

`monitor-checker` が銘柄チェック時に付与するタグ。
判定結果が **OK** であっても、注目すべき状況変化があるときに設定される。

```
OK + リスクフラグなし → 通知なし（静かにスルー）
OK + リスクフラグあり → 【確認】通知
NG                   → 【緊急】【警告】【朗報】通知（フラグはNG理由の補足情報）
```

## フラグ一覧

### ネガティブ系（懸念材料）

| フラグ | 意味 | 発動条件の目安 |
|--------|------|----------------|
| `price_deviation_exceeded` | 価格乖離超過（下落） | プラン時点から許容範囲を超えて下落（短期±3%、中期±5%、長期±10%） |
| `basis_invalidated` | 根拠崩壊 | プラン策定時の主要根拠（`decision_basis`）の前提が崩れた |
| `new_regulatory_risk` | 規制リスク | 新たな規制・制裁・法的リスクが報道された |
| `earnings_miss` | 決算未達 | 市場予想を下回る決算結果 |
| `sector_downturn` | セクター悪化 | 銘柄が属するセクター全体が悪化傾向 |
| `macro_shock` | マクロショック | 金利・為替・景気後退など広域の経済ショック |
| `management_change` | 経営陣変更 | CEO・CFO などの予期しない交代 |

### ポジティブ系（好材料・プラン見直しサイン）

| フラグ | 意味 | 発動条件の目安 |
|--------|------|----------------|
| `price_surge_exceeded` | 価格乖離超過（上昇） | プラン時点から許容範囲を超えて上昇（利確・売却タイミングの再検討が必要） |
| `target_reached` | 目標価格到達 | 設定した目標価格に到達した |
| `earnings_beat` | 決算好調 | 市場予想を上回る好決算 |
| `concern_resolved` | 懸念材料の解消 | 様子見判断の根拠だった懸念が払拭された（買い直し検討のサイン） |

## 補足

- フラグは `monitor-checker` が WebSearch で確認した事実にのみ基づいて設定される（推測では付与しない）
- 1回のチェックで複数フラグが同時に設定されることもある
- フラグの判定閾値（許容価格変動率など）は `monitor-checker.md` の判定基準セクションで管理している
