# ブロック別データフロー

各ブロックが何を受け取り、何を判断に使い、何を出力するかの一覧。

---

## 全体フロー図

```
Technical → Monitor → Analyzer → Planning → Watch
    ↓           ↓          ↓          ↓         ↓
 archive    archive    archive    archive   watchlist
.technical  .monitor   .lanes     .newplan   + Discord
                       .final_judge  _full   + action_log
```

---

## 1. Technical

| 区分 | 内容 |
|------|------|
| **入力** | watchlist（対象銘柄）、config.yaml（時間足・期間の設定） |
| **外部API** | yfinance（株価・テクニカル指標・為替レート） |
| **処理** | 銘柄ごとにテクニカル指標を取得。archive レコードを新規作成 |
| **出力** | `archive.technical`（最新株価、トレンド、モメンタム、ボラティリティ等） |

### 出力の主要フィールド

- `latest_price`：最新終値
- `usd_jpy_rate`：為替レート
- `timeframes.1d.indicators.raw`：テクニカル指標の生の数値（SMA、RSI、MACD 等 22種以上）
- `timeframes.1d.indicators.derived`：raw の数値を株の一般的な基準でラベルに変換したもの（例：RSI=55 → `rsi_state: "neutral"`）

**Monitor / Analyzer への渡し方**：raw（数値）と derived（ラベル）の両方を渡す（Issue #137 で最適化を検討予定）

## 2. ImportantIndicators

### 入力

- watchlist（対象銘柄）
- archive（前回の指標データ。連続失敗検知のみに使用。同じフィールドが2回連続 null なら Discord 通知）
- 外部API：yfinance（VIX、利回り、EPS、出来高等）、FRED API（政策金利）

### 処理

市場全体データを1回取得し、銘柄ごとに個別データを取得する。

### 出力

`archive.important_indicators` に以下を書き込む：

| フィールド | 内容 |
|-----------|------|
| `market` | VIX、米10年債利回り、FRB金利、日銀金利 |
| `event_risk` | 直近イベントまでの日数、Implied Move |
| `earnings` | EPS 予想 vs 実績、サプライズ% |
| `relative_strength` | 指数対比・セクター対比（3ヶ月） |
| `volume` | 5日平均出来高比、売買代金 |

## 3. Monitor

| 区分 | 内容 |
|------|------|
| **入力** | archive.newplan_full（前回プラン）、archive.technical、archive.important_indicators |
| **処理** | プランの前提が現在も有効かを WebSearch + テクニカル + 重要指標でチェック |
| **出力** | `archive.monitor`（OK/NG/ERROR、現在株価、変動率、サマリー、リスクフラグ）、`archive.MotivationID`（0=OK, 1=NG）、`archive.motivation_full`（NG理由）、`archive.active`（OK→False, NG→True）、`archive.status` |

### プロンプトに渡しているもの（build_check_prompt）

| 情報 | 渡している | 渡していない |
|------|:---------:|:-----------:|
| 判定（verdict） | o | |
| 投資期間（horizon） | o | |
| 確信度（confidence） | o | |
| 判定根拠（decision_basis） | o | |
| プラン時株価 | o | |
| テクニカル指標（derived ラベル + raw 数値） | o | |
| 重要指標（VIX、金利等） | o | |
| 損切り/利確ライン | o | |
| 許容価格変動幅 | o | |
| 投入額/株数 | o | |
| 保有状況（holdings） | o | |
| プランの鮮度 | o | |


### 出力の主要フィールド

- `result`：OK / NG / ERROR
- `current_price`：チェック時の株価
- `price_change_pct`：プラン時からの変動率
- `summary`：状況サマリー
- `risk_flags`：リスクフラグ配列
- `ng_reason`：NG の場合の理由

---

## 4. Analyzer

| 区分 | 内容 |
|------|------|
| **入力** | archive.technical、archive.monitor、archive.important_indicators、portfolio_config（レーン数等） |
| **処理** | 複数レーンで Analyst ↔ DA の議論 → Opinion → Judge → Final Judge |
| **出力** | `archive.lanes`（各レーンの議論ログ）、`archive.final_judge`（最終判定）、`archive.verdict` |
| **参照しない** | holdings（保有状況）、newplan_full（前回プラン） |

### portfolio_config から読む設定

- `analyzer_num_lanes`：並列レーン数（デフォルト 2）
- `analyzer_max_rounds`：議論ラウンド数（デフォルト 4）
- `analyzer_opinions_per_lane`：各レーンの意見数（デフォルト 2）

### 出力の主要フィールド

**archive.final_judge**：
- `decision`：BUY / SELL / HOLD / NO_BUY / NOT_BUY_WAIT 等
- `vote_for` / `vote_against`：賛成/反対票数
- `decision_basis`：根拠（lane + ソース + 理由）

**archive.lanes**：
- 各レーンの `discussion_md`（議論テキスト）、`judge_md`（判定テキスト）、`agreement`（AGREED/DISAGREED）

---

## 5. Planning

| 区分 | 内容 |
|------|------|
| **入力** | archive.final_judge、archive.lanes、archive.technical、archive.monitor、archive.important_indicators、portfolio_config（予算・リスク設定等）、holdings（保有状況） |
| **処理** | 鮮度チェック → 価格ズレ判定 → confidence 算出 → ポジションサイジング（既保有分考慮）→ 配分計算（残り予算）→ RR比 → YAML 生成 |
| **出力** | `archive.newplan_full`（プラン YAML、holdings セクション含む）、`archive.verdict`、`archive.status` |

### portfolio_config から読む設定

| 設定 | 用途 |
|------|------|
| `total_budget_jpy` | 総予算 |
| `risk_limit_pct` | 1銘柄あたりの最大損失許容% |
| `stop_loss_pct` | 損切り幅（期間別） |
| `default_take_profit_pct` | 利確ライン |
| `max_allocation_pct` | confidence 別の配分上限% |
| `price_tolerance_pct` | 許容価格変動幅（期間別）※判定には未使用、記録のみ |
| `price_block_pct` | プラン停止閾値（10%超で quantity=0） |
| `max_log_age_days` | ログ鮮度上限（期間別） |
| `min_rr_ratio` | 最低 RR 比（下回ると quantity=0） |

### 出力の主要ブロック（newplan_full）

| ブロック | 内容 |
|---------|------|
| `decision` | 判定、票数、confidence、投資期間、判定根拠 |
| `freshness` | ログ鮮度（日数、上限、OK/STALE） |
| `data_checks` | 基準価格、現在株価、価格ズレ%、許容幅、停止閾値 |
| `risk_management` | 許容損失額、ポジションサイズ上限、損切り/利確%、RR比 |
| `portfolio_plan` | 総予算、配分%、配分額、株数、為替、ロットサイズ |
| `execution_plan` | 注文方式、エントリールール、注意事項 |
| ~~`monitoring_hint`~~ | 廃止済み（Issue #136） |

---

## 6. Watch

| 区分 | 内容 |
|------|------|
| **入力** | archive.newplan_full、archive.verdict、archive.monitor、watchlist |
| **処理** | サマリー生成 → watchlist 更新 → Discord 通知 → action_log 投入 |
| **出力** | watchlist テーブル更新、Discord 通知、action_log テーブル投入、archive.active=False |

### watchlist に書き込むフィールド

- `MotivationID`：動機分類
- `motivation_summary`：NG 理由
- `discussion_result`：判定結果
- `discussion_summary`：議論サマリー
- `new_plan_summary`：新プラン概要
- `risk_flags`：リスクフラグ
- `plan_comparison`：前回プランとの比較
- `stock_price`：現在株価

| **損切り/利確ライン** | - | o（実装済み） | - | 自分が作成 | - |
| **投入額/株数** | - | o（実装済み） | - | 自分が作成 | - |

x = 参照すべきだが現在参照していない（Issue #134, MyDocs/各ブロック不足入力情報.md 参照）
